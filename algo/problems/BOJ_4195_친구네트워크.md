> [!info] 문제 정보
> - **플랫폼**: BOJ
> - **번호**: 4195
> - **티어**: Gold 2
> - **링크**: https://www.acmicpc.net/problem/4195

## 문제 요약

이름(String)으로 주어지는 두 사람을 친구로 추가할 때마다, 두 사람이 속한 친구 네트워크 크기를 출력한다.

## 접근법

> [!tip] 핵심 접근
> Union-Find + HashMap으로 String을 int ID로 매핑. parents[] 음수값으로 집합 크기 추적

### 사고 흐름

1. 이름 → 정수 ID 매핑: `HashMap<String, Integer>` + 등장 순서로 ID 부여
2. Union-Find의 parents[] 패턴 2 (음수 크기): `parents[root] < 0`, 절댓값 = 집합 크기
3. union(a, b) 후 두 사람이 속한 집합 크기 = `-parents[findSet(a)]`

### 핵심 코드

```java
static int[] parents;
static Map<String, Integer> map;
static int id;

static int getId(String name) {
    if (!map.containsKey(name)) {
        map.put(name, id);
        parents[id] = -1; // 크기 1짜리 독립 집합
        id++;
    }
    return map.get(name);
}

static int findSet(int a) {
    if (parents[a] < 0) return a;
    return parents[a] = findSet(parents[a]);
}

static int union(int a, int b) {
    int pA = findSet(a), pB = findSet(b);
    if (pA == pB) return -parents[pA]; // 이미 같은 집합
    if (parents[pA] <= parents[pB]) {
        parents[pA] += parents[pB];
        parents[pB] = pA;
        return -parents[pA];
    } else {
        parents[pB] += parents[pA];
        parents[pA] = pB;
        return -parents[pB];
    }
}
```

## 복잡도

| 항목 | 복잡도 | 근거 |
|------|--------|------|
| 시간 | O(F × α(N)) | 추정: F = 친구 관계 수, α(N) ≈ O(1) |
| 공간 | O(N) | 추정: parents 배열 + HashMap |

## 배운 점 / 인사이트

> [!note] 기억할 것
> - **String → int 매핑**: HashMap으로 이름을 ID로 변환, 문자열 Union-Find의 표준 패턴
> - **음수 크기 저장**: `parents[root] = -집합크기`. union 후 `return -parents[findSet(a)]`로 즉시 크기 반환
> - **parents 배열 크기**: 최대 등장 가능한 이름 수로 넉넉하게 (친구 관계 수 × 2 이하)
> - **테스트케이스 초기화**: 여러 TC에서 map, id, parents 모두 초기화 필수
