> - **번호**: 2117
> - **난이도**: D5 (추정)

## 문제 요약

N×N 격자에 집들이 있다. 서비스 영역 크기 K는 중심에서 맨해튼 거리 `K-1` 이내의 마름모 영역이며, 비용은 `K² + (K-1)²`. 각 집은 요금 M을 납부. 수익이 0 이상(`M × 집 수 ≥ 비용`)인 조건에서 서비스 가능한 **최대 집 수**를 구하라.

## 접근법

> [!tip] 핵심 접근
> 모든 N² 칸을 서비스 센터 후보로 완전탐색. 각 중심점에서 맨해튼 거리 분포를 int[] 배열로 집계하고, 거리 오름차순 누적합으로 최적 K를 O(N) 탐색.

### 사고 흐름

1. 서비스 영역 K의 비용 = `K² + (K-1)²` → 미리 `int[] costArr`로 계산
2. 모든 칸 (y, x)에 대해 모든 집까지의 맨해튼 거리를 `int[] distCount`에 집계
3. `distCount`를 거리 0부터 오름차순으로 누적 → 거리 d 이하의 집 수 = K=d+1 서비스 범위
4. `M × houses ≥ costArr[d+1]` 이면서 `houses > answer`이면 갱신

### 핵심 코드

```java
int[] costArr = new int[2 * N + 2];
for (int i = 1; i <= 2 * N + 1; i++) {
    costArr[i] = i * i + (i - 1) * (i - 1);
}

int[] distCount = new int[2 * N + 10]; // TC 내 재사용

for (int y = 0; y < N; y++) {
    for (int x = 0; x < N; x++) {
        Arrays.fill(distCount, 0);
        for (int h = 0; h < homeCount; h++) {
            int d = Math.abs(homeY[h] - y) + Math.abs(homeX[h] - x);
            distCount[d]++;
        }
        int houses = 0;
        for (int d = 0; d <= 2 * N; d++) {
            if (houses == homeCount) break;       // 이미 전체 포함, cost만 증가
            houses += distCount[d];
            if (M * houses >= costArr[d + 1] && houses > answer) {
                answer = houses;
            }
        }
        if (answer == homeCount) break;           // 전체 최대값 달성
    }
    if (answer == homeCount) break;
}
```

## 복잡도

| 항목 | 복잡도 | 근거 |
|------|--------|------|
| 시간 | O(N² × H) | N² 중심점 × H 집 수 거리 계산 (추정) |
| 공간 | O(N) | distCount, costArr 배열 |

## 최적화 히스토리 (버전별)

| 버전 | 방식 | 문제점 |
|------|------|--------|
| v1 | `TreeMap<Integer,Integer>[][]` | N² TreeMap 객체 생성, O(log H) 삽입 |
| v2 | `int[] dist` 단일 배열 재사용 | `static int[] dist = new int[2*N+1]` — N=0 버그 |
| v3 (목표) | `int[] homeY, homeX` 원시 배열 | Pair 객체 제거, 캐시 친화적 접근 |

## 배운 점 / 인사이트

> [!note] 기억할 것
> - **static 필드 초기화 함정**: `static int[] arr = new int[2*N+1]`은 클래스 로딩 시 N=0이므로 `new int[1]`이 됨 → 반드시 TC 루프 내 N 확정 후 할당
> - **TreeMap vs int[]**: 거리 범위가 `2*(N-1)`로 고정일 때 TreeMap은 과도한 설계. int[] + 인덱스로 O(1) 대체
> - **원시 배열 캐시 효율**: `List<Pair>` 순회는 힙 객체 역참조로 캐시 미스 발생. `int[] homeY, homeX` 연속 메모리 접근이 수십 배 빠름
> - **Early break 두 곳**: inner(거리 루프) + outer(중심점 루프) 각각 `homeCount` 도달 시 조기 종료
