> [!info] 문제 정보
> - **플랫폼**: BOJ
> - **번호**: 2098
> - **티어**: Gold 1
> - **링크**: https://www.acmicpc.net/problem/2098

## 문제 요약

N개 도시를 한 번씩 방문하고 출발 도시로 돌아오는 최소 비용 경로를 구한다. (TSP, Travelling Salesman Problem)

## 접근법

> [!tip] 핵심 접근
> `dp[visited][curr]` = `visited` 비트마스크 집합을 방문하고 현재 `curr`에 있을 때 최소 비용

### 사고 흐름

1. 순열 완전탐색은 O(N!) → N=16이면 2조, 불가
2. 방문 집합을 비트마스크로 표현 → 상태 수 N × 2^N
3. Bottom-up: `visited`를 0에서 (1<<N)-1 순으로 채워나감
4. 시작: `dp[1<<0][0] = 0` (0번 도시, 0번만 방문)
5. 전이: curr에서 미방문 next로 이동 → `dp[visited|(1<<next)][next] = min(..., dp[visited][curr] + cost[curr][next])`
6. 정답: `dp[(1<<N)-1][curr] + cost[curr][0]`의 최솟값 (모두 방문 후 0으로 복귀)

### 핵심 코드

```java
static final int INF = Integer.MAX_VALUE / 2;
static int[][] dp;
static int[][] W;
static int N;

// Bottom-up TSP
dp = new int[1 << N][N];
for (int[] row : dp) Arrays.fill(row, INF);
dp[1][0] = 0;

for (int visited = 1; visited < (1 << N); visited++) {
    for (int curr = 0; curr < N; curr++) {
        if (dp[visited][curr] == INF) continue;
        if ((visited & (1 << curr)) == 0) continue;
        for (int next = 0; next < N; next++) {
            if ((visited & (1 << next)) != 0) continue;
            if (W[curr][next] == 0) continue;
            int nv = visited | (1 << next);
            dp[nv][next] = Math.min(dp[nv][next], dp[visited][curr] + W[curr][next]);
        }
    }
}

int full = (1 << N) - 1;
int ans = INF;
for (int curr = 1; curr < N; curr++) {
    if (dp[full][curr] != INF && W[curr][0] != 0)
        ans = Math.min(ans, dp[full][curr] + W[curr][0]);
}
```

## 복잡도

| 항목 | 복잡도 | 근거 |
|------|--------|------|
| 시간 | O(N² × 2^N) | 추정: 상태 N×2^N, 각 상태에서 N개 전이 |
| 공간 | O(N × 2^N) | 추정: dp 배열 |

N = 16 → 약 1,048,576 × 16 = 16M 상태, 충분히 가능

## 배운 점 / 인사이트

> [!note] 기억할 것
> - **시작 노드 고정**: 출발점을 0번으로 고정해도 TSP의 최적해 불변 (순환이므로)
> - **복귀 처리 분리**: 마지막 전이에서 0번으로의 복귀는 루프 외부에서 따로 처리
> - **INF 오버플로우**: `Integer.MAX_VALUE + anything = 음수` → 반드시 `/2` 처리
> - **경로 없음 = 0**: `W[i][j] == 0`이면 경로 없음으로 스킵
