> [!info] 문제 정보
> - **플랫폼**: BOJ
> - **번호**: 13460
> - **티어**: Gold 1
> - **링크**: https://www.acmicpc.net/problem/13460

## 문제 요약

N×M 보드에서 빨간 구슬을 구멍에 빠뜨리되 파란 구슬은 빠뜨리지 않아야 한다. 기울이기(4방향)를 최소 횟수로 수행한다. 10번 초과 시 -1.

## 접근법

> [!tip] 핵심 접근
> BFS + 4D 방문 배열 `vis[redY][redX][blueY][blueX]`로 중복 상태 제거

### 사고 흐름

1. 상태 = 빨간 구슬 위치 + 파란 구슬 위치 = 4차원
2. BFS로 기울이기 횟수(레벨) 순서로 탐색 → 최소 횟수 보장
3. 기울일 때 두 구슬 모두 이동 처리 필요:
   - 같은 방향으로 굴릴 때 **선행 구슬 먼저 이동** (한쪽이 다른 쪽을 막음)
   - 예: 오른쪽으로 기울이면 x가 더 큰 구슬 먼저
4. 파란 구슬이 먼저 빠지면 실패
5. 빨간 구슬이 빠지고 파란이 안 빠지면 성공
6. 10번 초과하면 -1

### 핵심 코드

```java
// 이동 처리: 방향 d로 구슬 굴리기
static int[] move(int y, int x, int dy, int dx, char[][] board) {
    while (board[y + dy][x + dx] != '#' && board[y][x] != 'O') {
        y += dy; x += dx;
    }
    return new int[]{y, x};
}

// BFS 큐 상태: {redY, redX, blueY, blueX, depth}
// 선행 구슬 결정 (오른쪽 기울이기일 때 x가 큰 구슬 먼저)
if (dx == 1) {
    if (rx > bx) { /* 빨강 먼저 */ } else { /* 파랑 먼저 */ }
}
```

## 복잡도

| 항목 | 복잡도 | 근거 |
|------|--------|------|
| 시간 | O(N² × M²) | 추정: 상태 수 (N×M)² = 최대 (10×10)² = 10,000 |
| 공간 | O(N² × M²) | 추정: 방문 배열 크기 |

## 배운 점 / 인사이트

> [!note] 기억할 것
> - **4D visited 배열**: 두 객체의 위치를 동시에 상태로 표현할 때 4차원 vis 배열 사용
> - **선행 구슬 처리**: 같은 방향으로 이동 시 앞서 있는 구슬부터 이동 → 뒤 구슬이 앞에 막히지 않도록
> - **구멍 처리**: 구슬이 구멍에 빠지면 이동 중단 (while 조건에 `board[y][x] != 'O'` 추가)
> - **파랑 먼저 확인**: 이동 후 파랑이 구멍이면 즉시 실패 처리 (빨강도 빠졌어도 실패)
