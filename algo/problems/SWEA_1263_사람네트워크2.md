> [!info] 문제 정보
> - **플랫폼**: SWEA
> - **번호**: 1263
> - **유형**: 그래프, 전체 쌍 최단경로

## 문제 요약

N명의 사람과 연결 관계(인접 행렬 형태)가 주어질 때, 한 사람을 선택해 나머지 모든 사람에게 연락하는 데 필요한 총 단계 수의 최솟값을 구한다.

## 접근법

> [!tip] 핵심 접근
> 플로이드 워셜로 모든 쌍 최단거리를 계산한 뒤, 각 노드의 거리 합 중 최솟값을 출력.

### 사고 흐름

1. 인접 행렬로 초기 그래프 구성 (연결 = 1, 미연결 = INF)
2. 무향 그래프이므로 `dist[u][v] = dist[v][u] = 1`
3. 플로이드 워셜로 전체 쌍 최단거리 계산
4. 각 노드 u에 대해 `Σ dist[u][v]` 합산 → 최솟값이 정답

### 핵심 코드

```java
// 올바른 루프 순서: k→i→j
for (int k = 1; k <= N; k++) {
    for (int i = 1; i <= N; i++) {
        for (int j = 1; j <= N; j++) {
            if (dist[i][k] != INF && dist[k][j] != INF) {
                dist[i][j] = Math.min(dist[i][k] + dist[k][j], dist[i][j]);
            }
        }
    }
}

// 거리 합 최솟값
long answer = INF;
for (int u = 1; u <= N; u++) {
    long sum = 0;
    for (int v = 1; v <= N; v++) {
        if (u != v) sum += dist[u][v];
    }
    answer = Math.min(answer, sum);
}
```

## 복잡도

| 항목 | 복잡도 | 근거 |
|------|--------|------|
| 시간 | O(N³) | 플로이드 워셜 |
| 공간 | O(N²) | `long[][]` dist 행렬 |

## 배운 점 / 인사이트

> [!note] 기억할 것
> - 거리 합 오버플로우 주의: `int` 대신 `long` 사용, INF는 `Long.MAX_VALUE / 2`
> - 입력이 인접 행렬 전체로 주어질 때는 `row × col` 이중 루프로 파싱
> - [[SWEA_5643_키순서]]와 구조 동일, 집계 방식(합산 vs 도달 가능 수)만 다름
