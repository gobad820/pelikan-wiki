> [!info] 문제 정보
> - **플랫폼**: SWEA
> - **번호**: 5643
> - **유형**: 그래프, 최단경로, 도달가능성

## 문제 요약

N명의 학생과 M개의 키 비교 결과(A < B)가 주어질 때, 자신의 키 순서를 정확히 알 수 있는 학생의 수를 구한다.

## 접근법

> [!tip] 핵심 접근
> 플로이드 워셜로 전이적 도달 가능성을 계산. i→j 도달 가능 = i가 j보다 키가 작음.

### 사고 흐름

1. 직접 비교만으론 전이 관계를 놓침 (A<B, B<C → A<C 를 추론해야 함)
2. 플로이드 워셜로 모든 쌍의 도달 가능 여부를 O(N³)에 계산
3. 노드 i 기준: `dist[i][j] != INF` (i가 j보다 작음) + `dist[j][i] != INF` (j가 i보다 작음) 개수 합산
4. 합이 N-1 이상이면 모든 다른 학생과 크기 비교 가능 → 순서 확정

### 핵심 코드

```java
// 플로이드 워셜 (k가 최외곽이어야 함)
for (int k = 1; k <= N; k++) {
    for (int i = 1; i <= N; i++) {
        for (int j = 1; j <= N; j++) {
            if (dist[i][k] != INF && dist[k][j] != INF) {
                dist[i][j] = Math.min(dist[i][k] + dist[k][j], dist[i][j]);
            }
        }
    }
}

// 키 순서 확정 가능 여부 판별
for (int i = 1; i <= N; i++) {
    int temp = 0;
    for (int j = 1; j <= N; j++) {
        if (i == j) continue;
        if (dist[i][j] != INF) temp++; // i가 j보다 작음
        if (dist[j][i] != INF) temp++; // j가 i보다 작음
    }
    if (temp >= N - 1) answer++;
}
```

## 복잡도

| 항목 | 복잡도 | 근거 |
|------|--------|------|
| 시간 | O(N³) | 플로이드 워셜 3중 루프 |
| 공간 | O(N²) | 인접 행렬 dist |

## 접근법 2 — DFS (정방향 + 역방향)

> [!tip] 핵심 접근
> 정방향 DFS(내가 도달 가능한 노드 = 나보다 키 작은 수) + 역방향 DFS(나에게 도달 가능한 노드 = 나보다 키 큰 수) 합산.

### 사고 흐름

1. `adj` (A→B: A<B) + `rAdj` (역방향) 두 그래프 구성
2. 노드 i에서 `dfs(i, adj)` → i보다 키 작은 학생 수
3. 노드 i에서 `dfs(i, rAdj)` → i보다 키 큰 학생 수
4. 합이 N-1이면 순서 확정

### 핵심 코드

```java
for (int i = 1; i <= N; i++) {
    Arrays.fill(vis, false);
    int temp = dfs(i, adj) - 1;     // 자기 자신 제외
    Arrays.fill(vis, false);
    temp += (dfs(i, rAdj) - 1);
    answer += temp == N - 1 ? 1 : 0;
}

static int dfs(int i, List<Integer>[] adj) {
    int result = 1;
    for (int next : adj[i]) {
        if (vis[next]) continue;
        vis[next] = true;
        result += dfs(next, adj);
    }
    return result;
}
```

### 복잡도 (DFS)

| 항목 | 복잡도 | 근거 |
|------|--------|------|
| 시간 | O(N × (V+E)) 추정 | 노드마다 정/역 DFS 1회 |
| 공간 | O(N+M) 추정 | 인접 리스트 2개 + vis 배열 |

> [!note] 실측 성능 (SWEA 제출 기준)
> 메모리 102,232 kb / 실행시간 2,181 ms

## 풀이 버전 비교

| 버전 | 파일 | 시간복잡도 | 비고 |
|------|------|-----------|------|
| v1 | 1BFS | O(N×(V+E)) | 디버그 출력 미제거 → WA |
| v2 | 2DFS | O(N×(V+E)) | 통과. `vis[startNode]` 미설정 (DAG 보장돼 문제없음) |
| v3 | 3플로이드워샬 | O(N³) | 버그 2개 수정 완료 |

## 버그 노트

> [!warning] 제출 전 확인할 버그
> - **1BFS**: `System.out.println` 디버그 출력 미제거 → WA
> - **3플로이드워샬** (수정 완료): 루프 순서 `i→j→k` 오류 + `answer` 미초기화
> - **2DFS 주의**: `vis[startNode]` 를 true로 설정하지 않음 → 사이클 있는 그래프에서 무한루프 위험 (이 문제는 DAG라 무관)

## 배운 점 / 인사이트

> [!note] 기억할 것
> - 전이적 관계(A<B<C → A<C)는 플로이드 워셜로 한 번에 해결 가능
> - 순서 확정 조건: "나보다 작은 애 수 + 나보다 큰 애 수 = N-1"
