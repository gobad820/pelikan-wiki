# 백준 1463 - 1로 만들기

> 정수 N을 아래 세 연산으로 1로 만드는 최소 연산 횟수
> - 3으로 나누기 (나누어 떨어질 때만)
> - 2로 나누기 (나누어 떨어질 때만)
> - 1 빼기

## 핵심 아이디어

**왜 완전탐색이 안 되는가?**  
각 단계에서 최대 3가지 선택 → 지수 시간 O(3ⁿ), N이 크면 불가.

**최적 부분 구조 성립?**  
i를 1로 만드는 최소 연산 = 한 번의 연산 후 남은 수를 최적으로 처리  
→ `f(i) = min(f(i/3)+1, f(i/2)+1, f(i-1)+1)`

## 점화식

```
f(i) = f(i-1) + 1                         // 항상 적용
if (i % 3 == 0): f(i) = min(f(i), f(i/3) + 1)
if (i % 2 == 0): f(i) = min(f(i), f(i/2) + 1)
```

## 기저값

```
f(1) = 0   // 이미 1이므로 연산 불필요
f(2) = 1   // 2÷2 = 1
f(3) = 1   // 3÷3 = 1
```

## Java 코드

```java
static final int MAXVAL = 1_001_000;
static int[] dp = new int[MAXVAL];

static {
    Arrays.fill(dp, Integer.MAX_VALUE);
    dp[1] = 0;
    dp[2] = 1;
    dp[3] = 1;
    for (int i = 4; i < MAXVAL; i++) {
        if (i % 3 == 0) dp[i] = Math.min(dp[i / 3] + 1, dp[i]);
        if (i % 2 == 0) dp[i] = Math.min(dp[i / 2] + 1, dp[i]);
        dp[i] = Math.min(dp[i - 1] + 1, dp[i]);
    }
}
// dp[N]이 정답
```

> `Integer.MAX_VALUE`로 초기화 → 각 연산 적용 가능할 때만 min 갱신

## 시간복잡도

- 전처리: O(N)
- 쿼리: O(1)

## 주의사항

- `i % 3`, `i % 2` 조건 체크를 **먼저** 하고 나서 `i - 1` 처리 순서 상관없음
- 처리 순서: 어떤 연산이든 `min`으로 갱신하므로 순서 무관
- `Integer.MAX_VALUE` + 1 → 오버플로우 위험이 있으나 dp[i-1]은 MAX가 아니므로 안전
- 초기값을 `dp[2]=1, dp[3]=1` 직접 설정해 반복문을 `i=4`부터 시작

## 패턴 특징: 다중 전이 DP

각 상태에서 전이 가능한 경우가 여러 개이고, **조건(나누어 떨어지는지)**에 따라 선택 가능 경우가 달라짐.
