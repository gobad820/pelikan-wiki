> [!info] 문제 정보
> - **플랫폼**: BOJ
> - **번호**: 2110
> - **티어**: Gold 4
> - **링크**: https://www.acmicpc.net/problem/2110

## 문제 요약

N개 집 위치에 C개 공유기를 설치할 때, 인접 공유기 사이의 최소 거리를 최대화한다.

## 접근법

> [!tip] 핵심 접근
> "최솟값의 최대화" → 파라메트릭 서치. 최소 거리 `d`를 이분탐색하며 `d` 이상으로 C개 설치 가능한지 체크

### 사고 흐름

1. "최솟값을 최대화"하는 전형적인 파라메트릭 서치 패턴 인식
2. 정답 공간: `[1, max_pos - min_pos]` (최소 거리 ~ 최대 거리)
3. `check(d)`: 집 위치 정렬 후 그리디하게 C개 설치 가능한지 확인
   - 첫 집에 설치, 이후 `현재 위치 - 마지막 설치 위치 >= d`이면 설치
4. `check(mid) == true` → 더 크게 시도 (`lo = mid + 1`, `ans = mid`)
5. `check(mid) == false` → 더 작게 시도 (`hi = mid - 1`)

### 핵심 코드

```java
static int[] pos;
static int N, C;

static boolean check(int minDist) {
    int cnt = 1, last = pos[0];
    for (int i = 1; i < N; i++) {
        if (pos[i] - last >= minDist) {
            cnt++;
            last = pos[i];
            if (cnt >= C) return true;
        }
    }
    return cnt >= C;
}

// main
Arrays.sort(pos);
int lo = 1, hi = pos[N-1] - pos[0], ans = 0;
while (lo <= hi) {
    int mid = (lo + hi) / 2;
    if (check(mid)) { ans = mid; lo = mid + 1; }
    else hi = mid - 1;
}
```

## 복잡도

| 항목 | 복잡도 | 근거 |
|------|--------|------|
| 시간 | O(N log D) | 추정: check O(N), 이분탐색 O(log D), D=최대 거리 범위 |
| 공간 | O(1) | 추정: 입력 배열 외 추가 없음 |

## 배운 점 / 인사이트

> [!note] 기억할 것
> - **단조성 확인**: `d`가 작을수록 설치 쉬움, 클수록 어려움 → `check`가 단조 감소 → 파라메트릭 서치 적용 가능
> - **정렬 필수**: check 함수의 그리디가 올바르려면 위치 배열 오름차순 정렬 필요
> - **ans 갱신 시점**: `check(mid) == true`일 때 `ans = mid` 저장 후 더 크게 탐색
> - **경계값**: `lo = 1` (최소 거리 1), `hi = pos[N-1] - pos[0]` (모든 집이 같은 선상 최대)
