> [!info] 문제 정보
> - **플랫폼**: Code Tree
> - **유형**: 시뮬레이션, 배열
> - **핵심**: N×N 보드에서 연속으로 같은 수가 M개 이상인 행 또는 열의 수를 구하라

## 문제 요약

N×N 정수 보드가 주어질 때, 각 행·열에서 연속으로 같은 값이 M개 이상 등장하는 행/열의 수를 합산한다.

## 접근법

> [!tip] 핵심 접근
> 슬라이딩 윈도우 대신 **연속 카운터(cnt)** 단일 순회로 O(N²) 해결

### 사고 흐름

1. M=1이면 모든 행·열이 조건 만족 → 답 = N×2
2. M>1이면 행 N개, 열 N개 각각 순회
3. 각 행/열에서 `cnt=1`로 시작, 이전과 같으면 `cnt++`, 다르면 `cnt=1` 리셋
4. `cnt >= M`이면 해당 행/열 카운트 후 `break`

### 핵심 코드

```java
if (M == 1) {
    answer = N * 2;
} else {
    for (int y = 0; y < N; y++) {       // 행 검사
        int cnt = 1;
        for (int x = 1; x < N; x++) {
            cnt = (board[y][x] == board[y][x - 1]) ? cnt + 1 : 1;
            if (cnt >= M) { answer++; break; }
        }
    }
    for (int x = 0; x < N; x++) {       // 열 검사
        int cnt = 1;
        for (int y = 1; y < N; y++) {
            cnt = (board[y][x] == board[y - 1][x]) ? cnt + 1 : 1;
            if (cnt >= M) { answer++; break; }
        }
    }
}
```

## 복잡도

| 항목 | 복잡도 | 근거 |
|------|--------|------|
| 시간 | O(N²) | 행 N회 × 열 N회 순회 |
| 공간 | O(N²) | 보드 배열 |

## 디버깅 히스토리 (오늘 발생한 버그)

| 버그 | 원인 | 교훈 |
|------|------|------|
| `isHappy` 항상 false | `true` 설정 코드 없음 | 플래그 변수 상태 추적 필수 |
| 윈도우 크기 M+1 | `i < M`이 M쌍 비교 = M+1원소 | 루프 종료 후 i값 직접 추적 |
| `continue` 사용 | 같은 행 중복 카운트 | 탈출은 `break`, 다음 이터레이션은 `continue` |
| `cnt = 0` 리셋 | M-1 연속까지 감지 불가 | 리셋은 반드시 `cnt = 1` |
| 마지막 윈도우 누락 | 루프 범위 `< N-M+1` | 경계값에 최솟값·최댓값 직접 대입 검증 |

## 배운 점 / 인사이트

> [!note] 기억할 것
> - 코드 수정 후 변경된 값이 **연관된 모든 조건에 반영됐는지** 체크 필수
> - 루프 범위는 **"첫 번째 경우"와 "마지막 경우"를 직접 대입**해 인덱스 검증
> - 중첩 루프(슬라이딩 윈도우)보다 단일 카운터가 구현 단순하고 버그 적음
