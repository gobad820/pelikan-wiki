# 백준 1010 - 다리 놓기

> 서쪽 N개 사이트를 동쪽 M개 사이트에 교차 없이 연결하는 경우의 수

## 핵심 아이디어

다리가 교차하지 않으려면 서쪽 i번째 사이트는 반드시 동쪽에서 i번째로 작은 사이트에 연결된다.  
→ M개 중 N개를 순서대로 고르면 됨 → **mCn 조합**

**왜 공식 직접 계산이 안 되는가?**  
29! 은 `long` 범위도 초과 (약 8.8 × 10³⁰). 오버플로우.

**왜 완전탐색이 안 되는가?**  
₂₉C₁₄ ≈ 7,700만, 시간 제한 0.5초 → 불가.

## 점화식

M번째 동쪽 사이트를 기준으로 분류:
- M번째 사용하는 경우 → `f(N-1, M-1)`
- M번째 사용하지 않는 경우 → `f(N, M-1)`

```
f(N, M) = f(N, M-1) + f(N-1, M-1)   // 파스칼 삼각형과 동일 구조
```

## 기저값

```
f(1, M) = M     // 1개 고르는 경우
f(N, N) = 1     // N개 중 N개 고르는 경우
```

## Bottom-Up (상향식) 코드

```java
static long[][] dp = new long[30][30];

static {
    for (int i = 1; i < 30; i++) dp[1][i] = i;     // f(1, M) = M
    for (int i = 2; i < 30; i++) {
        dp[i][i] = 1;                               // f(N, N) = 1
        for (int j = i + 1; j < 30; j++) {
            dp[i][j] = dp[i][j - 1] + dp[i - 1][j - 1]; // 점화식
        }
    }
}
// dp[N][M] = mCn
```

> `static {}` 블록에서 클래스 로딩 시 전처리 → 쿼리마다 O(1)

## Top-Down (하향식) 코드

```java
static long getTheNumberOfBridges(int left, int right) {
    if (left == right) return dp[left][right] = 1;
    if (left == 1)     return dp[left][right] = right;
    if (dp[left][right] != 0) return dp[left][right];  // 캐시 히트
    return dp[left][right] =
        getTheNumberOfBridges(left, right - 1) +
        getTheNumberOfBridges(left - 1, right - 1);
}
```

> 초기값이 0이므로 `dp[n] != 0`으로 캐시 체크 (일반적인 -1 대신)

## 주의사항

- `int` → 오버플로우. `long` 사용 필수
- 상향식에서 `j`는 반드시 `i+1`부터 시작 (dp[i][i]는 이미 1로 설정)
- 파스칼 삼각형과 인덱스 의미가 다름: `dp[N][M]` = C(M, N) = mCn
