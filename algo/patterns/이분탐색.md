> [!abstract] 핵심 요약
> 정렬된 탐색 공간에서 O(log N) 탐색. 파라메트릭 서치(매개변수 탐색)는 "가능/불가능 이분"으로 최적값을 구하는 기법.

## 언제 쓰나

- **기본 이분탐색**: 정렬된 배열에서 특정 값 위치 찾기 O(log N)
- **파라메트릭 서치**: "X가 가능한가?" 함수가 단조성(monotone)을 만족할 때 최적값 탐색
  - "최댓값의 최솟값", "최솟값의 최댓값" 형태 문제
  - 예: 공유기 설치 (최소 거리의 최대화), 나무 자르기 (최소 높이의 최대화)
- 정렬 후 lower_bound / upper_bound 활용 (LIS O(N log N))

## 핵심 아이디어

### 기본 이분탐색
```
[lo, hi] 범위를 반씩 좁혀 target 위치 찾기
mid = (lo + hi) / 2
arr[mid] < target → lo = mid + 1
arr[mid] > target → hi = mid - 1
arr[mid] == target → 발견
```

### 파라메트릭 서치 (Parametric Search)
```
answer 공간 [lo, hi]에서 이분탐색
check(mid) = "mid가 정답이 될 수 있는가?"
단조성: check(x)가 true면 check(x-1)도 true (또는 반대)
→ true/false 경계를 이분탐색으로 찾음
```

## 복잡도

| 항목 | 복잡도 | 비고 |
|------|--------|------|
| 시간 | O(N log M) | 추정: N=check 비용, M=탐색 범위 |
| 공간 | O(1) | 추정: 추가 배열 불필요 |

## Java 코드 템플릿

### 기본 이분탐색

```java
// 정렬된 arr에서 target의 위치 (없으면 -1)
static int binarySearch(int[] arr, int target) {
    int lo = 0, hi = arr.length - 1;
    while (lo <= hi) {
        int mid = lo + (hi - lo) / 2; // 오버플로우 방지
        if (arr[mid] == target) return mid;
        else if (arr[mid] < target) lo = mid + 1;
        else hi = mid - 1;
    }
    return -1;
}
```

### Lower Bound (target 이상인 첫 위치)

```java
static int lowerBound(int[] arr, int target) {
    int lo = 0, hi = arr.length;
    while (lo < hi) {
        int mid = (lo + hi) / 2;
        if (arr[mid] < target) lo = mid + 1;
        else hi = mid;
    }
    return lo; // arr[lo] >= target인 첫 인덱스
}
```

### 파라메트릭 서치 - "최솟값의 최대화" (BOJ 2110 공유기 패턴)

```java
// 공유기 설치: 인접 공유기 사이 최소 거리의 최대값
static int[] pos; // 정렬된 위치 배열
static int C;     // 공유기 개수

static boolean check(int minDist) {
    int count = 1;
    int last = pos[0];
    for (int i = 1; i < pos.length; i++) {
        if (pos[i] - last >= minDist) {
            count++;
            last = pos[i];
        }
    }
    return count >= C;
}

static int solve() {
    Arrays.sort(pos);
    int lo = 1;
    int hi = pos[pos.length - 1] - pos[0];
    int ans = 0;
    while (lo <= hi) {
        int mid = (lo + hi) / 2;
        if (check(mid)) {
            ans = mid;   // mid가 가능 → 더 크게 시도
            lo = mid + 1;
        } else {
            hi = mid - 1; // mid 불가능 → 더 작게
        }
    }
    return ans;
}
```

## 주의사항 / 자주 하는 실수

> [!warning] 실수 포인트
> - **무한루프**: `lo = mid`, `hi = mid` 로 하면 lo==hi일 때 무한루프. 반드시 `mid+1` 또는 `mid-1`
> - **오버플로우**: `(lo + hi) / 2` 대신 `lo + (hi - lo) / 2` 사용
> - **단조성 확인**: check(x)가 단조 증가/감소인지 먼저 확인. 아니면 파라메트릭 서치 불가
> - **정렬 전처리**: 파라메트릭 서치에서 배열을 미리 정렬해야 함
> - **경계값**: `lo = 1`, `hi = max_possible`로 설정 (0이나 max+1 포함 주의)
> - **최대화/최소화 방향**: 가능하면 ans 갱신 후 `lo = mid + 1` (최대화), `hi = mid - 1` (최소화)
