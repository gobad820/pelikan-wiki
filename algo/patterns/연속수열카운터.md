> [!abstract] 핵심 요약
> 1차원 배열에서 연속된 동일 원소 구간을 O(N)에 탐지하는 패턴.
> 슬라이딩 윈도우(O(N·M)) 대신 카운터 하나로 해결한다.

## 언제 쓰나

- 배열에서 **연속으로 같은 값이 M개 이상** 등장하는 구간 탐지
- 행·열 단위로 조건을 만족하는 행/열 **개수 카운트**
- 최장 연속 동일 구간 길이 구하기

## 핵심 아이디어

```
cnt = 1  (현재 원소 1개로 시작)
순회하며:
  현재 == 이전  →  cnt++
  현재 != 이전  →  cnt = 1  (새 구간 시작)
  cnt >= M      →  조건 만족 → 처리 후 break
```

> [!warning] cnt 리셋 시 1로 초기화
> `cnt = 0`으로 리셋하면 다음 비교에서 같아도 cnt=1이 되어 실제 2개 연속을 1개로 인식 → M-1개까지 감지 불가 버그 발생.
> 반드시 `cnt = 1`로 리셋해야 한다.

## 복잡도

| 항목 | 복잡도 | 비고 |
|------|--------|------|
| 시간 | O(N) | 1회 순회 |
| 공간 | O(1) | 카운터 변수 1개 |

슬라이딩 윈도우(중첩 루프 O(N·M)) 대비 상수 배 빠름.

## Java 코드 템플릿

```java
// 행 검사 — M개 이상 연속 동일 원소가 있는 행 카운트
for (int y = 0; y < N; y++) {
    int cnt = 1;
    for (int x = 1; x < N; x++) {
        cnt = (board[y][x] == board[y][x - 1]) ? cnt + 1 : 1;
        if (cnt >= M) { answer++; break; }
    }
}

// 열 검사
for (int x = 0; x < N; x++) {
    int cnt = 1;
    for (int y = 1; y < N; y++) {
        cnt = (board[y][x] == board[y - 1][x]) ? cnt + 1 : 1;
        if (cnt >= M) { answer++; break; }
    }
}
```

## 슬라이딩 윈도우(중첩 루프) vs 연속 카운터 비교

| 구분 | 슬라이딩 윈도우 | 연속 카운터 |
|------|----------------|------------|
| 복잡도 | O(N·M) | O(N) |
| 구현 난이도 | 높음 (경계 조건 오류 잦음) | 낮음 |
| Off-by-one 위험 | 높음 (윈도우 시작/끝) | 낮음 |
| 권장 | 합계 등 값 변화 추적 필요 시 | 동일 값 연속 탐지 시 |

## 주의사항 / 자주 하는 실수

> [!warning] 실수 포인트
> - `cnt = 0`으로 리셋 → M-1 연속까지 감지 불가 (반드시 `cnt = 1`)
> - 루프 변수 `i`의 최종값과 조건 `i == M` 불일치 — 루프가 `i < M-1`로 끝나면 완료 시 `i = M-1`
> - `continue` 사용 → 같은 행/열이 중복 카운트됨 (조건 만족 즉시 `break`)
> - 경계 인덱스: 윈도우 마지막 원소가 N-1에 닿는지 대입 검증 필수
