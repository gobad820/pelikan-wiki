> [!abstract] 핵심 요약
> 원소를 키(점수·등급) 별 버킷(링크드 리스트)에 분산 저장해 일괄 변경을 O(1)로 처리. 삭제는 version 번호로 Lazy 처리해 fire O(1) 달성.

## 언제 쓰나

- **"팀 전체 점수 변경" + "개별 삭제/수정"** 연산이 혼재하고, 조회 횟수가 적을 때
- 원소가 유한한 범위의 키(점수, 등급 등)로 분류 가능할 때
- SDS 시험 시뮬레이션 문제에서 `hire / fire / updateTeam / bestMember` 형태의 API 구현

## 핵심 아이디어

1. `bucket[팀][점수]`: 각 (팀, 점수) 조합마다 더미 헤드(sentinel) + 테일 포인터를 가진 링크드 리스트
2. **일괄 변경 = 체인 이동**: 점수 j → k로 바뀌면 `tail[k].next = head[j].next`로 j 버킷 전체를 k 버킷 뒤에 O(1) 연결
3. **Lazy 무효화**: 삭제 시 `version[id] = -1`. 조회 시 `node.value == version[node.id]`로 유효성 체크
4. **재삽입 = 업데이트**: 점수 변경 시 새 버전으로 새 버킷에 append → 기존 노드는 자동 무효

## 복잡도

| 연산 | 복잡도 | 비고 |
|------|--------|------|
| hire | O(1) | tail append |
| fire | O(1) | version 갱신만 |
| updateSoldier | O(1) | hire 재호출 |
| updateTeam | O(점수 범위) ≈ O(1) | 버킷 수만큼 체인 이동 |
| bestSoldier | O(버킷 내 노드 수) | 무효 노드 포함 순회 — 호출 횟수가 적어서 허용 |

## Java 코드 템플릿

```java
// 노드 풀 + 버킷 구조
static class Node {
    int id, value; // value = 생성 시 version 번호
    Node next;
}
static Node[] pool = new Node[MAX_POOL];
static int cnt = 0;

static int[] version = new int[MAX_ID]; // version[id] = 현재 유효 버전
static int[] team    = new int[MAX_ID]; // team[id] = 소속 팀

// bucket[t][s]: 팀 t, 점수 s의 버킷
static Node[][] head = new Node[6][6]; // 더미 sentinel
static Node[][] tail = new Node[6][6];

static Node getNewNode(int id) {
    Node n = pool[cnt++];
    n.id = id;
    n.value = ++version[id]; // 새 버전 부여
    n.next = null;
    return n;
}

// 초기화
static void init() {
    cnt = 0;
    for (int i = 0; i < MAX_POOL; i++) pool[i] = new Node();
    for (int t = 1; t <= 5; t++)
        for (int s = 1; s <= 5; s++)
            tail[t][s] = head[t][s] = getNewNode(0); // 더미 헤드
    // version, team 배열 초기화
}

// hire: O(1)
static void hire(int id, int t, int s) {
    Node n = getNewNode(id);
    tail[t][s].next = n;
    tail[t][s] = n;
    team[id] = t;
}

// fire: O(1) Lazy
static void fire(int id) { version[id] = -1; }

// updateSoldier: O(1) — 재삽입
static void updateSoldier(int id, int newScore) { hire(id, team[id], newScore); }

// updateTeam: O(점수 범위) — 체인 병합
static void updateTeam(int t, int delta) {
    if (delta > 0) {
        for (int j = 5; j >= 1; j--) {           // 높은 점수부터
            int k = Math.min(5, j + delta);
            if (j == k || head[t][j].next == null) continue;
            tail[t][k].next = head[t][j].next;    // 체인 연결
            tail[t][k]      = tail[t][j];          // tail 갱신
            head[t][j].next = null;                // j 버킷 초기화
            tail[t][j]      = head[t][j];
        }
    } else if (delta < 0) {
        for (int j = 1; j <= 5; j++) {             // 낮은 점수부터
            int k = Math.max(1, j + delta);
            if (j == k || head[t][j].next == null) continue;
            tail[t][k].next = head[t][j].next;
            tail[t][k]      = tail[t][j];
            head[t][j].next = null;
            tail[t][j]      = head[t][j];
        }
    }
}

// bestSoldier: O(버킷 크기) — 호출 횟수 적으므로 허용
static int bestSoldier(int t) {
    for (int j = 5; j >= 1; j--) {
        int ans = 0;
        Node cur = head[t][j].next;
        while (cur != null) {
            if (cur.value == version[cur.id])
                ans = Math.max(ans, cur.id);
            cur = cur.next;
        }
        if (ans != 0) return ans;
    }
    return 0;
}
```

## 주의사항 / 자주 하는 실수

> [!warning] 실수 포인트
> - **updateTeam 루프 방향**: 점수 **올리기**는 j=5→1 (높은 것부터), **내리기**는 j=1→5 (낮은 것부터)
>   → 이미 이동한 버킷을 다시 이동하지 않기 위해
> - **sentinel 더미 헤드**: `head[t][s]`는 더미 노드 → `head[t][s].next == null`이 빈 버킷
> - **team[] 배열**: updateSoldier에서 소속 팀 재조회를 위해 hire 시 `team[id] = t` 저장 필수
> - **version 초기화**: 매 TC에서 `version[]` 전체 초기화 필수 (이전 TC 잔여값 영향)
> - **pool 크기**: hire 횟수 + 더미 헤드 수 (팀×점수 = 25개) + 여유분
