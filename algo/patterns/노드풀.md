> [!abstract] 핵심 요약
> 링크드 리스트 노드를 `new`로 동적 생성하지 않고, 시작 전에 배열로 미리 할당해두고 인덱스로 꺼내 쓰는 패턴. GC 부담 없이 안정적인 실행 시간 확보.

## 언제 쓰나

- 링크드 리스트 노드를 **반복적으로 생성**해야 하는 시뮬레이션 문제
- SDS / SWEA 시험처럼 **엄격한 시간 제한**에서 `new` 비용을 제거해야 할 때
- 여러 테스트케이스에서 노드를 재사용해야 할 때 (`cnt = 0`으로 풀 리셋)
- [[버킷링크드리스트]] 패턴과 함께 사용할 때

## 핵심 아이디어

1. 프로그램 시작 시 `pool[MAX]` 배열에 노드를 **딱 한 번만** `new`로 생성
2. 노드가 필요할 때마다 `pool[cnt++]`으로 꺼냄 — `new` 호출 없음
3. TC 초기화 시 `cnt = 0`으로 리셋 → 기존 노드 객체를 재사용

## 복잡도

| 항목 | 복잡도 | 비고 |
|------|--------|------|
| getNewNode() | O(1) | 배열 인덱스 접근만 |
| init() 리셋 | O(1) | cnt = 0만 변경 (재초기화 불필요) |
| 메모리 | O(MAX_POOL) | 사전 고정 할당 |

## Java 코드 템플릿

```java
static class Node {
    int value;
    Node next;
}

static final int MAX_POOL = 100_005; // 최대 노드 생성 수 + 여유
static Node[] pool = new Node[MAX_POOL];
static int cnt = 0;

// 프로그램 시작 시 or static 블록에서 한 번만 호출
static void initPool() {
    for (int i = 0; i < MAX_POOL; i++) pool[i] = new Node();
}

// TC 초기화: cnt만 0으로 리셋 (new 없이 재사용)
static void init() {
    cnt = 0;
    head = null;
    tail = null;
    listSize = 0;
}

// 노드 꺼내기: O(1)
static Node getNewNode(int value) {
    Node n = pool[cnt++];
    n.value = value;
    n.next = null;
    return n;
}
```

### 풀 크기 계산 방법

```
MAX_POOL = 초기 원소 수 + 삽입 연산 최대 횟수 + 여유(1.2배)
예) 초기 N=1000, 삽입 M=1000회, TC=10 → 10 × 2000 × 1.2 = 24,000
```

## 주의사항 / 자주 하는 실수

> [!warning] 실수 포인트
> - **풀 크기 부족**: 넉넉하게 잡지 않으면 ArrayIndexOutOfBoundsException → 여유 1.2~2배 권장
> - **init()에서 pool 재생성 금지**: `cnt = 0`만 해도 충분. `for (new Node()...)` 반복하면 오히려 느림
> - **노드 필드 초기화**: `getNewNode()`에서 `n.next = null` 반드시 설정 (이전 TC 잔여값)
> - **정적 초기화 위치**: `initPool()`은 main() 진입 직후 단 한 번만 호출
