# 2026-04-08 강의 노트

## 오늘 배운 패턴/개념
- [[LIS]] — 최장 증가 부분 수열: DP O(N²) 기본형 + 이진탐색 O(N log N) 고속형
- [[Knapsack_01]] — 0/1 배낭 문제: 각 물건을 한 번만 선택, DP 2D → 1D 공간 최적화

## 핵심 코드

### LIS — DP O(N²)

```java
int[] lis = new int[N];
int max = 0;
for (int i = 0; i < N; i++) {
    lis[i] = 1;
    for (int j = 0; j < i; j++) {
        if (arr[j] < arr[i] && lis[i] < lis[j] + 1) {
            lis[i] = lis[j] + 1;
        }
    }
    max = Math.max(max, lis[i]);
}
```

### 0/1 Knapsack — 2D 배열 (직관적)
```java
for (int i = 1; i <= N; i++) {
    for (int j = 0; j <= K; j++) {
        dp[i][j] = (V[i] > j)
            ? dp[i - 1][j]
            : Math.max(dp[i - 1][j], dp[i - 1][j - V[i]] + C[i]);
    }
}
```

### 1D 배열 (공간 최적화 — 역순 순회 핵심)
```java
for (int i = 1; i <= N; i++) {
    for (int j = K; j >= V; j--) {
        dp[j] = Math.max(dp[j], dp[j - V] + C);
    }
}
```

## 문제 풀이
- **LIS 기본 구현**: `LIS[i]` = arr[i]를 끝으로 하는 최장 증가 부분 수열 길이. O(N²) DP.
- **SWEA 3282 (0/1 Knapsack)**: 2D → 1D 두 가지 풀이 모두 구현. 1D 역순 순회의 이유 이해가 핵심.
- **SWEA 5215 (햄버거다이어트)**: 칼로리 = 무게, 맛 = 가치로 변수를 치환하면 그대로 Knapsack 틀 적용.

## 오늘의 인사이트
- LIS의 핵심 정의: **`LIS[i]`는 "i번째 원소를 반드시 포함하는" LIS 길이**. "1부터 i까지의 최장"이 아님에 주의.
  - 따라서 최종 답은 `max(LIS[0..N-1])` — 마지막 원소가 아니라 전체 최댓값.
- LIS O(N log N): `tails[]` 배열이 항상 정렬 상태를 유지하기 때문에 이진탐색 가능. 입력 배열 정렬과는 무관.
  - `tails[]`는 실제 LIS 수열이 아니라 "길이별 최솟값 꼬리" 보조 배열임을 혼동하지 말 것.
- 1D 배열에서 **역순 순회**하는 이유: `dp[j - V]`가 아직 현재 물건 i를 고려하지 않은 값이어야 중복 선택을 막을 수 있음. 순방향이면 같은 아이템이 여러 번 들어가 Unbounded Knapsack이 됨.
- 문제에서 "각 재료는 최대 1번 사용"이라는 조건이 보이면 0/1 Knapsack 신호.
- Knapsack은 패턴 자체보다 **변수 매핑** (무엇이 무게? 무엇이 가치?)이 풀이의 핵심.
