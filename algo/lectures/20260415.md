## 오늘 학습한 패턴/개념

- [[KMP]] — pi 배열(실패 함수) 기반 O(N+M) 단일 패턴 검색
- [[RabinKarp]] — 롤링 해시 기반 O(N+M) 평균 패턴 검색. 다중 패턴 확장 용이

## 핵심 코드 스니펫

### KMP — pi 배열 구성 패턴

```java
// buildPi: i=1부터 시작, 불일치 시 j = pi[j-1]로 점프
static int[] buildPi(String p) {
    int m = p.length();
    int[] pi = new int[m];
    int j = 0;
    for (int i = 1; i < m; i++) {
        while (j > 0 && p.charAt(i) != p.charAt(j)) j = pi[j - 1];
        if (p.charAt(i) == p.charAt(j)) pi[i] = ++j;
    }
    return pi;
}
```

## 오늘의 인사이트

> [!quote] 핵심 기억
> KMP의 `buildPi`는 패턴을 텍스트로, 패턴을 패턴으로 KMP 검색하는 구조. pi 배열 자체가 KMP를 자신에 적용한 결과다. `i=1` 시작과 일치 후 `j = pi[j-1]` 점프가 핵심.

> [!note] KMP vs Rabin-Karp 선택 기준
> - 단일 패턴, 안정성 중요 → **KMP** (O(N+M) 항상 보장)
> - 다중 패턴, 구현 단순성 → **Rabin-Karp** (HashSet 확장, 이중 해시로 충돌 완화)
