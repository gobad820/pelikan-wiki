## 오늘 배운 패턴/개념
- Dijkstra - PriorityQueue 기반 다익스트라 O(E log V), 두 가지 스킵 패턴

## 핵심 코드
```java
// PQ에 넣을 정보 (정점 번호 + 현재까지 거리)
static class Vertex implements Comparable<Vertex> {
    int no, dist;
    @Override public int compareTo(Vertex o) { return Integer.compare(this.dist, o.dist); }
}

Arrays.fill(minDist, Integer.MAX_VALUE);
minDist[start] = 0;
PriorityQueue<Vertex> pq = new PriorityQueue<>();
pq.offer(new Vertex(start, 0));

while (!pq.isEmpty()) {
    Vertex cur = pq.poll();
    if (visited[cur.no]) continue;       // 패턴 1: visited로 중복 스킵
    // if (cur.dist > minDist[cur.no]) continue; // 패턴 2: dist 비교로 스킵
    visited[cur.no] = true;

    for (Node temp = adjList[cur.no]; temp != null; temp = temp.next) {
        if (!visited[temp.to] && minDist[temp.to] > cur.dist + temp.weight) {
            minDist[temp.to] = cur.dist + temp.weight;
            pq.offer(new Vertex(temp.to, minDist[temp.to]));
        }
    }
}
```

## 문제 풀이
- **Dijkstra_AdjList_PQ**: visited[] 방식
- **Dijkstra_AdjList_PQ2**: dist 비교 방식 (visited 배열 없이)
- **Solution_1249_보급로**: 0-1 BFS 또는 Dijkstra 적용 (가중치 = 복구 시간)

## 오늘의 인사이트
- PQ 방식: O(E log V) — 같은 정점이 PQ에 여러 번 들어갈 수 있으므로 반드시 스킵 처리
- 두 스킵 패턴 차이:
  - `visited[]`: 명시적으로 방문 표시, 직관적
  - `cur.dist > minDist[cur.no]`: 배열 추가 없이 가능
- PQ에 넣는 시점: `minDist` 갱신될 때마다 → 동일 정점이 여러 번 들어가도 무방
