## 오늘 학습한 패턴/개념

- [[플로이드워셜]] — 모든 쌍 최단경로, 3중 루프(k→i→j), O(N³)
- [[Backtracking]] — 완전탐색 + 가지치기, 스킵 우선 탐색으로 효율 극대화
- [[BFS_DFS]] — 정방향 DFS + 역방향 DFS로 도달 가능성 집계

## 핵심 코드 스니펫

```java
// 플로이드 워셜 — k가 반드시 최외곽
for (int k = 1; k <= N; k++)
    for (int i = 1; i <= N; i++)
        for (int j = 1; j <= N; j++)
            if (dist[i][k] != INF && dist[k][j] != INF)
                dist[i][j] = Math.min(dist[i][j], dist[i][k] + dist[k][j]);
```

## 오늘의 인사이트

> [!quote] 핵심 기억
> 플로이드 워셜에서 루프 순서는 k→i→j. k(경유 노드)가 확정된 상태에서 i→j를 갱신해야 "k까지만 경유" 의미가 유지된다. i→j→k로 쓰면 k가 아직 확정 안 된 상태에서 갱신이 일어나 오답이 된다.
