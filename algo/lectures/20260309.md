## 오늘 배운 패턴/개념
- [[UnionFind]] - 서로소 집합 자료구조, 경로 압축 + 랭크 기반 union

## 핵심 코드
```java
static int[] parents, ranks;

static void makeSets(int N) {
    parents = new int[N + 1];
    ranks = new int[N + 1];
    for (int i = 0; i <= N; i++) parents[i] = i;
}

// 경로 압축: 재귀 후 parents[a]를 루트로 직접 연결
static int findSet(int a) {
    if (a == parents[a]) return a;
    return parents[a] = findSet(parents[a]);
}

// 랭크 기반 union: 트리 높이 최소화
static boolean union(int a, int b) {
    int rootA = findSet(a), rootB = findSet(b);
    if (rootA == rootB) return false; // 이미 같은 집합 → 사이클
    if (ranks[rootB] > ranks[rootA])       parents[rootA] = rootB;
    else if (ranks[rootB] < ranks[rootA])  parents[rootB] = rootA;
    else { parents[rootB] = rootA; ranks[rootA]++; }
    return true;
}
```

## 문제 풀이
- **마을 그룹 수 (TheNumberOfVillageGroup)**: union 후 루트가 다른 집합 수 카운트
- **서로소 집합 (CoprimeSet)**: union-find로 같은 집합 여부 판단
- **도시 도로망 (CityAndRoadNetwork)**: 연결 컴포넌트 수 계산
- **파핑파핑 지뢰찾기**: BFS + 연결 컴포넌트

## 오늘의 인사이트
- `findSet` 에서 `parents[a] = findSet(parents[a])` — 재귀 반환 시 경로 전체를 루트에 직접 연결 (경로 압축)
- `union` 반환 `false` = 이미 같은 집합 = **사이클 존재** → Kruskal MST에서 핵심 사용
- 집합 수 = N - (union 성공 횟수)
- 랭크는 트리 높이의 상한, 경로 압축 이후엔 실제 높이와 다를 수 있음
