## 오늘 배운 패턴/개념
- [[MST_Prim]] - 정점 중심으로 최소 신장 트리 확장 (인접 리스트 버전)

## 핵심 코드
```java
// Prim - 인접 리스트 버전 (연결 리스트 노드 구조)
static Node[] adjList;   // 인접 리스트
static int[] minEdge;    // 각 정점까지 MST 연결 최소 비용
static boolean[] visited;

Arrays.fill(minEdge, Integer.MAX_VALUE);
minEdge[0] = 0; // 임의의 시작 정점

int result = 0;
for (int c = 0; c < V; c++) {
    // Step1: 비트리 정점 중 minEdge 최소인 정점 선택
    int min = Integer.MAX_VALUE, minVertex = -1;
    for (int i = 0; i < V; i++) {
        if (!visited[i] && min > minEdge[i]) {
            minVertex = i;
            min = minEdge[i];
        }
    }
    if (minVertex == -1) break;
    visited[minVertex] = true;
    result += min;

    // Step2: 새로 추가된 정점의 인접 정점 minEdge 갱신
    for (Node temp = adjList[minVertex]; temp != null; temp = temp.next) {
        if (!visited[temp.vertex] && minEdge[temp.vertex] > temp.weight) {
            minEdge[temp.vertex] = temp.weight;
        }
    }
}
```

## 문제 풀이
- **최소 스패닝 트리 (Solution_3124)**: Prim으로도 동일 결과
- **하나로 (Solution_1251)**: Prim 적용 (밀집 그래프에 유리)

## 오늘의 인사이트
- Prim은 `minEdge[]` 배열로 각 정점이 MST에 연결되는 최소 비용 유지
- `minEdge[0] = 0` 설정으로 시작 정점 강제 선택 (비용 0 → 첫 번째 Step1에서 무조건 선택됨)
- 인접 리스트 버전 vs 행렬 버전: 희소 그래프는 인접 리스트가 효율적
- 행렬 버전(MST_Prim_Matrix): 이중 for문으로 인접 확인, O(V²)
- Prim O(V²) vs Kruskal O(E log E) — 밀집 그래프면 Prim, 희소 그래프면 Kruskal 유리
