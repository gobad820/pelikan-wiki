## 오늘 배운 패턴/개념
- [[Greedy]] - 회의실 배정 (Activity Selection Problem)

## 핵심 코드
```java
class Meeting implements Comparable<Meeting> {
    int start, end;
    @Override
    public int compareTo(Meeting o) {
        if (o.end == this.end) return this.start - o.start;
        return this.end - o.end; // 끝나는 시간 오름차순
    }
}

List<Meeting> getSchedule(Meeting[] meetings) {
    Arrays.sort(meetings);       // 종료시간 오름차순
    List<Meeting> result = new ArrayList<>();
    result.add(meetings[0]);
    for (int i = 1; i < meetings.length; i++) {
        if (result.get(result.size() - 1).end <= meetings[i].start)
            result.add(meetings[i]);
    }
    return result;
}
```

## 오늘의 인사이트
- 그리디 증명: 끝이 가장 빠른 회의 선택 → 다음 회의 기회 최대화
- `Comparable<T>` 구현: `compareTo`에서 음수=this가 앞, 양수=o가 앞
- `end <= start`: 같을 때 배정 가능 (등호 포함)
