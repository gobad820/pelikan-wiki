## 오늘 학습한 패턴/개념

- [[연속수열카운터]] — 행·열에서 연속 동일 원소 M개 이상 탐지, O(N) 단일 카운터
- [[도형완전탐색]] — 트로미노 6방향 완전 열거, 경계 조건 삼항 연산자 처리

## 핵심 코드 스니펫

### 연속 카운터 패턴

```java
// 행 검사
for (int y = 0; y < N; y++) {
    int cnt = 1;
    for (int x = 1; x < N; x++) {
        cnt = (board[y][x] == board[y][x-1]) ? cnt + 1 : 1;  // 리셋은 1
        if (cnt >= M) { answer++; break; }
    }
}
```

### 도형 완전 탐색 패턴

```java
// 경계 조건을 삼항 연산자로 처리
int s0 = (x+1 >= M || y-1 < 0) ? 0
       : board[y][x] + board[y][x+1] + board[y-1][x];
answer = Math.max(answer, s0);
```

## 핵심 개념 정리

### 연속 카운터 vs 슬라이딩 윈도우

| 구분 | 슬라이딩 윈도우 | 연속 카운터 |
|------|----------------|------------|
| 복잡도 | O(N·M) | O(N) |
| Off-by-one 위험 | 높음 | 낮음 |
| 권장 상황 | 합계 변화 추적 | 동일 값 연속 탐지 |

### 트로미노 도형 6방향

```
L자형 (4방향)      직선형 (2방향)
  ┘ ┌ └ ┐            ─  │
```

### 디버깅 체크리스트 (오늘 반복된 실수 유형)

| 실수 유형 | 증상 | 예방법 |
|----------|------|--------|
| 플래그 미설정 | 조건 항상 false | 플래그 true 설정 코드 존재 여부 확인 |
| 루프 종료값 불일치 | i==M 조건이 참 안 됨 | `i < M-1` 루프는 종료 시 `i=M-1` |
| break/continue 혼용 | 중복 카운트 | 탈출=break, 다음 반복=continue |
| cnt=0 리셋 | M-1개까지만 감지 | 리셋은 반드시 cnt=1 |
| 배열 크기 N×N | N≠M 케이스 AIOOBE | board = new int[N][M] |

## 트러블슈팅 기록

- `isHappy`가 항상 false → true 설정 코드 없음 → 플래그 패턴 사용 시 true/false 양방향 확인
- 루프 `i < M-1` 후 `i == M` 체크 → 종료값 불일치 → 수정 후 연관 조건 전부 재검토
- `cnt = 0` 리셋 → `[1,2,1,1,1]` M=3 케이스에서 cnt 최대 2 → `cnt = 1`로 수정
- `board = new int[N][N]` → 열 크기 M 초과 접근 시 AIOOBE → `new int[N][M]`

## 오늘의 인사이트

> [!quote] 핵심 기억
> 코드를 수정할 때는 **변경된 값이 연관된 모든 조건에 반영됐는지** 체인처럼 따라가야 한다.
> 루프 변수의 최종값은 루프 조건식에서 직접 계산한다 (`i < M-1` → 종료 시 `i = M-1`).
> 연속 동일 원소 탐지는 중첩 루프 없이 **단일 카운터** 하나로 충분하다 — 복잡한 구조를 먼저 떠올리지 말 것.
