## 정의

원소를 **키(점수, 등급 등) 별 버킷(링크드 리스트)**에 분산 저장해 특정 키의 원소를 O(1)로 일괄 이동/병합하는 구조. 삭제는 실제로 리스트에서 제거하지 않고 **버전(version) 번호로 유효성을 표시**하는 Lazy 방식으로 처리.

## 구조

```
Team[t].head[score]  ←→  ...노드... → tail[score]
         head[1] → [id=3,ver=2] → [id=7,ver=1] → null
         head[2] → null
         head[3] → [id=5,ver=3] → null
         ...
```

- `t[팀][점수]`: 더미 헤드(sentinel) + 테일 포인터를 가진 링크드 리스트
- 노드: `{id, value(버전), next}`
- `version[id]`: 해당 id의 현재 유효 버전 번호
- `node.value == version[node.id]` → 유효한 노드

## 핵심 연산

### hire (삽입): O(1)
```java
// 해당 팀, 해당 점수 버킷의 tail에 append
t[mTeam].tail[mScore].next = newNode;
t[mTeam].tail[mScore] = newNode;
```

### fire (삭제): O(1) — Lazy
```java
// 실제 리스트에서 제거하지 않고 버전 무효화
version[mID] = -1;
```

### updateSoldier (점수 변경): O(1)
```java
// 새 버전으로 새 점수 버킷에 재삽입 → 기존 노드는 자동 무효화
hire(mID, num[mID], newScore);
// num[mID] = 현재 소속 팀 (hire 시 저장)
```

### updateTeam (팀 전체 점수 변경): O(점수 범위) = O(5)
```java
// 점수 올리기: 높은 점수 버킷부터 순서대로 체인 병합
if (mChangeScore > 0) {
    for (int j = 5; j >= 1; j--) {
        int k = Math.min(5, Math.max(1, j + mChangeScore));
        if (j == k || head[j].next == null) continue;
        // j 버킷 전체를 k 버킷 tail에 연결 (O(1) 체인 이동)
        tail[k].next = head[j].next;
        tail[k] = tail[j];
        head[j].next = null;
        tail[j] = head[j]; // 빈 버킷으로 리셋
    }
}
// 점수 내리기: 낮은 점수 버킷부터 순서대로 (순서 반대)
```

### bestSoldier (최고 병사 조회): O(점수 범위 × 버킷 크기)
```java
for (int j = 5; j >= 1; j--) {
    // j 버킷에서 유효한 노드 중 최대 id 탐색
    int ans = 0;
    Node cur = head[j].next;
    while (cur != null) {
        if (cur.value == version[cur.id])  // 유효 체크
            ans = Math.max(ans, cur.id);
        cur = cur.next;
    }
    if (ans != 0) return ans;
}
```

## Lazy 무효화가 가능한 이유

- **bestSoldier 호출 횟수가 적다** (문제 조건 최대 100회)
- 무효 노드가 쌓여도 조회 시 `version` 체크로 걸러냄
- 반면 hire/fire/update는 최대 100,000회 → 이쪽은 O(1) 필수

## 알고리즘과의 관계

- [[노드풀]] — 버킷 내 노드도 풀에서 꺼내 쓰면 GC 최소화
- SDS 시뮬레이션 문제에서 "팀 전체 일괄 변경 + 개별 삭제/수정"이 나오면 이 패턴 적용 검토

## updateTeam 루프 방향 주의사항

> [!warning] 체인 이동 방향
> - 점수 **올리기** (`+`): **높은 버킷 → 낮은 버킷** 순으로 처리 (j=5→1)
>   → 이미 이동한 버킷을 다시 이동하지 않도록
> - 점수 **내리기** (`-`): **낮은 버킷 → 높은 버킷** 순으로 처리 (j=1→5)
>   → 동일한 이유
