## 정의

링크드 리스트의 노드를 `new` 키워드로 동적 할당하는 대신, **미리 배열로 노드를 생성해두고** 인덱스로 꺼내 쓰는 기법.

```java
static Node[] pool = new Node[MAX];  // 풀 미리 생성
static int cnt = 0;                  // 다음 꺼낼 위치

static Node getNewNode(int value) {
    Node cur = pool[cnt++];  // new 없이 기존 객체 재사용
    cur.value = value;
    cur.next = null;
    return cur;
}
```

## 핵심 특성

- **GC 부담 없음**: 객체를 재사용하므로 GC 호출 최소화 → 시간 안정성
- **init() 패턴**: TC 초기화 시 `cnt = 0` 리셋 + 모든 노드 재초기화
- **풀 크기 계산**: 연산 횟수 × 노드 생성 횟수로 상한 추정 후 여유롭게 잡음
- **단방향 소비**: 풀에서 꺼낸 노드는 반납하지 않음. `cnt`만 0으로 리셋하면 재사용 가능

## 사용 패턴

```java
// 노드 클래스
static class Node {
    int value;
    Node next;
}

// 풀 선언 (전역 static)
static Node[] pool = new Node[MAX_SIZE];
static int cnt = 0;

// 반드시 init에서 미리 생성
static void init() {
    cnt = 0;
    for (int i = 0; i < MAX_SIZE; i++) {
        pool[i] = new Node();  // 딱 한 번만 new
    }
    // 리스트 head/tail 초기화 ...
}

// 노드 꺼내기
static Node getNewNode(int value) {
    Node n = pool[cnt++];
    n.value = value;
    n.next = null;
    return n;
}
```

## add / delete / get 구현 시 주의점

```java
// add(index, value): index 위치 앞에 삽입
// index == 0 → head 앞에 삽입 (head 교체)
// index == listSize → tail 뒤에 삽입
// 그 외 → (index-1)번째 노드까지 순회 후 삽입

// delete(index): index 위치 노드 제거
// index == 0 → head = head.next
// 그 외 → (index-1)번째 노드의 next를 next.next로 교체

// get(index): O(N) 순회, tail 캐싱으로 마지막 접근 최적화 가능
```
