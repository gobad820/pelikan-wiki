## 정의

클라이언트(브라우저)의 요청을 받아 처리하고 응답을 돌려주는 서버 구조.
정적 처리를 담당하는 **Web Server**와 동적 처리를 담당하는 **WAS**로 구분된다.

---

## Web Server vs WAS

| 구분 | 역할 | 예시 |
|------|------|------|
| Web Server | 정적 파일(HTML, CSS, JS, 이미지) 그대로 반환 | Apache, Nginx |
| WAS | 코드를 실행해 동적으로 응답 생성 | Tomcat, JBoss |

```
[Web Server]
브라우저 → 요청 → 웹 서버 → 파일 탐색 → 응답 → 브라우저

[WAS]
브라우저 → 요청 → WAS → 코드 실행(동적 처리) → 응답 → 브라우저
```

---

## 웹 서버의 한계

```
POST /login HTTP/1.1
Content-Type: application/x-www-form-urlencoded

id=mino&password=1234
```

위 요청에서 실제로 필요한 처리:
1. `id=mino`, `password=1234` 값 파싱
2. DB에서 사용자 조회
3. 인증 성공 시 로그인 처리

→ 웹 서버는 `/login` 파일을 찾아 반환할 뿐, **동적 처리 불가**

## HTTP 요청/응답 기본 구조

### Request

```
GET /index.html HTTP/1.1        ← 요청 라인 (메서드, 경로, HTTP 버전)
Host: 192.168.204.100:5500      ← 요청 헤더
Accept: text/html
                                ← 빈 줄 (헤더 끝)
                                ← 바디 (GET은 비어 있음)
```

### Response

```
HTTP/1.1 200 OK                 ← 상태 라인 (HTTP 버전, 상태 코드, 메시지)
Content-Type: text/html         ← 응답 헤더
Content-Length: 2572
                                ← 빈 줄
<!DOCTYPE html>                 ← 바디 (실제 응답 데이터)
<html>...</html>
```

| 구성 요소 | Request | Response |
|-----------|---------|----------|
| 첫 번째 줄 | 요청 라인 (`메서드 경로 HTTP버전`) | 상태 라인 (`HTTP버전 상태코드 메시지`) |
| 헤더 | Host, Accept, Content-Type ... | Content-Type, Content-Length ... |
| 빈 줄 | 헤더 끝 구분 (필수) | 헤더 끝 구분 (필수) |
| 바디 | POST 요청에서 사용 | HTML, JSON, 이미지 등 |

---

## WAS — Tomcat

앞으로 사용할 WAS. Java 코드를 실행할 수 있는 WAS.

```
브라우저 → 요청 → Tomcat(WAS) → Servlet 실행(Java 코드) → 응답 → 브라우저
```

Tomcat이 개발자 코드를 실행하려면 **약속된 형식**이 필요 → 그 약속이 **Servlet**

---

## Servlet과의 관계

[[Servlet]]은 Tomcat이 실행할 수 있도록 약속된 형식으로 작성된 Java 파일.
Tomcat은 요청을 받으면 URL에 매핑된 Servlet을 찾아 실행한다.

```
브라우저 → 요청 → Tomcat → URL 매핑 → Servlet 실행 → 응답
```

> [!note] 핵심 관계
> Web Server ⊂ WAS: Tomcat은 Web Server 기능도 포함하지만,
> 실무에서는 앞단에 Nginx(Web Server)를 두고 뒤에 Tomcat(WAS)을 배치하는 구조가 일반적.

---

## 주의사항 / 자주 하는 실수

> [!warning] 실수 포인트
> - Web Server와 WAS를 혼동 — 정적/동적 처리 담당이 다름
> - Tomcat은 WAS이면서 Web Server 기능도 수행하지만, 완전한 Web Server는 아님
> - HTTP 빈 줄(CRLF)은 헤더와 바디 구분에 **필수** — 없으면 파싱 오류
